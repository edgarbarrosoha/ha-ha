La clave: La mayoría de esas conexiones potenciales no se activan porque la gente no sabe lo que los demás tienen o necesitan. El sistema que proponemos activa esas conexiones de forma inteligente.

La Base de Conocimiento: el cerebro del sistema

La base de conocimiento no es una hoja de cálculo. Es un sistema semántico — entiende relaciones, no solo texto.

Pregunta Base de datos tradicional Base de conocimiento semántica
"¿Quién sabe de logística?" Busca la palabra "logística" en perfiles Encuentra a quien dirige distribución, hizo tesis de supply chain, exporta productos — aunque ninguno escribió "logística"
"¿Hay oportunidades cruzadas?" No puede responder "3 empresas de manufactura necesitan distribución en el Bajío, y 2 de logística buscan clientes industriales"

Los 4 Agentes de Inteligencia

Cada agente genera un tipo distinto de valor. Juntos, forman un ciclo de inteligencia que se fortalece con cada vuelta.

Match — El Conector

Encuentra oportunidades de negocio entre miembros mediante matching anonimizado. El 80% de los miembros buscan esto.

Contenidos — El Curador

Personaliza capacitación según perfil individual. Genera demanda informada de programas de Educación Continua.

Scout — El Cazatalentos

Detecta candidatos para futuras generaciones analizando redes profesionales de miembros actuales.

Insights — El Estratega

Genera inteligencia estratégica para el Consejo. De decidir por intuición a decidir con evidencia.

Resumen ejecutivo para Jorge Blando

¿Dónde viven los datos?
En infraestructura del Tec (Azure)
¿Quién es dueño?
Tec Beyond — 100% de los datos. HA solo de su código.
¿HA puede llevarse datos?
No. Si se termina el contrato, datos quedan en el Tec.
¿Los miembros están protegidos?
Sí: consentimiento, anonimización, derechos ARCO, auditoría.
¿Qué estándares de seguridad?
Encriptación AES-256, TLS 1.3, MFA, backup automático.
¿Cumple con la ley mexicana?
Sí: LFPDPPP, derechos ARCO, aviso de privacidad.

Resumen para Jorge Blando

¿Ya tenemos el business plan?
No — tenemos el framework. Los números se construyen juntos en sesiones de trabajo.
¿Cuánto cuesta?
Se define en sesión de descubrimiento. Estructura: inversión inicial + operación mensual + costo marginal por sede.
¿Cuándo se paga sola?
Meta: durante Fase 2 (transición), cuando membresías + fuentes ≥ costos de operación.
¿Cuál es el riesgo financiero?
Acotado: el piloto es la inversión mayor. Cada sede adicional cuesta una fracción. Si no funciona, datos quedan.
¿Hay ROI no financiero?
Sí: diferenciación, retención, datos para EC, posicionamiento AI del Tec, pipeline de learners.

Modelo de Escalabilidad

Cómo crece Tec Beyond de León a una red nacional, cuánto cuesta cada paso, y qué debe comprobarse antes de expandir.

El principio: fractal, no lineal

La plataforma se construye una vez en León. Cada sede nueva hereda todo y solo configura lo local: comunidad, contexto regional, administración.

Criterios Go/No-Go para expandir

Negocio

≥30 empresarios interesados, Consejo local activo, admin confirmado, presupuesto aprobado.

Técnico

99.5% uptime, matches aceptados ≥40%, datos migrados al 100%, admin entrenado.

Red

Sedes existentes en verde, ≥10 matches inter-sede demostrados.

Si no se cumple

Se posterga. No se fuerza. Mejor una sede menos que una sede débil.

Resumen para Jorge Blando

¿Cómo escala?
Fractal: la plataforma se construye una vez y se replica. Cada sede hereda todo, solo configura lo local.
¿Cuánto cuesta cada sede nueva?
~20-25% del costo del piloto. La plataforma ya existe.
¿Cuántas sedes por año?
Máximo 2, con criterios de go/no-go antes de cada una.
¿Cuándo se activa el matching entre sedes?
Con la segunda sede (Expansión 1). Ahí se desbloquea el efecto de red.
¿Cuál es la visión a 10 años?
~8,000 miembros en 8-10 sedes. La comunidad empresarial más sofisticada de México. Replicable a otros programas del Tec.
