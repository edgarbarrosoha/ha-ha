## El Core: TaaS + MARA

GPT existia antes de ChatGPT. El modelo era poderoso pero inaccesible. El punto de inflexion fue la INTERFAZ.

### TaaS (Thinking as a Service) — El Motor

| Capacidad | Descripcion |
|---|---|
| 6 dimensiones como API | Cada dimension es un endpoint consultable |
| V = M x I computarizable | La formula de valor se vuelve calculo, no metafora |
| Matriz de interaccion | 6 x 5 = 30 interacciones entre dimensiones |
| Generacion de conocimiento | Desde memoria de 3 niveles |
| Propiedad fractal | Cualquier escala: persona, proyecto, organizacion |

### MARA — La Interfaz (Punto de Inflexion)

| Capacidad | Descripcion |
|---|---|
| Primera interfaz que consume TaaS | Demuestra que el motor funciona |
| Usuarios: humanos Y agentes de IA | Doble audiencia desde el inicio |
| Minimalista + complejidad bajo demanda | Simple por default, profundo cuando se necesita |
| Sentirse como PENSAR, no como operar | UX orientado a cognicion, no a features |
| La demostracion viva del sistema | MARA es el proof-of-concept permanente |

## Datos vs. Conocimiento en TaaS

Los datos son del cliente. El conocimiento generado es de todos.

### Intencion como ancla

HA captura la intencion con el Legado. Todo dato y todo conocimiento existen porque hubo una intencion que les dio sentido.

```
Legado → Intencion → Datos → Conocimiento
```

### La distincion

| | Datos | Conocimiento |
|---|---|---|
| **Propiedad** | Privados — del cliente | Perdura — sobrevive al proyecto |
| **Portabilidad** | Removibles — el cliente los lleva | Se comparte — beneficia sin exponer datos |
| **Ciclo de vida** | Olvidables — se pueden borrar | Crece — cada interaccion lo alimenta |

### Pipeline de memoria

```
Working Memory (sesion) → Project Memory (persistente) → Organizational Memory (anonimizado, compartido)
```
