# Context: ha-score

## Landscape
- **GitHub Contribution Graph** — Temporal activity heatmap, single dimension (commits)
- **Gantt charts** — Task timelines, no cross-dimensional view
- **Obsidian Activity** — Plugin-level, no multi-vault, no dimension awareness
- **Personal analytics** — RescueTime, Toggl: measure time, not creative output
- **Musical notation software** — Sibelius, Dorico: for actual music, not data

## Positioning
HA Score is unique because:
1. Multi-vault, multi-dimensional temporal visualization
2. Musical notation metaphor is native (not forced) — Edgar IS a composer
3. Narrative movements give semantic structure to time
4. Cross-dimension barlines reveal simultaneous complexity
5. Self-contained, no infrastructure

## Constraints
- macOS-only for now (stat birthtime syntax)
- Git required for ha-eb (only vault with version control)
- Date range currently hardcoded (Jan-Mar 2026, 5 movements)
- Description quality depends on folder naming conventions

## Opportunities
- **Anagrama presentation** — Live score as book artifact (deadline: Mar 11)
- **JIII paper** — Visual evidence for notation systems argument
- **Consulting** — "Here's your organization's score" as diagnostic tool
- **AI composition** — An agent that proposes what the next bars should contain
