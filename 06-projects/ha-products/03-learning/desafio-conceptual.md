## Definición

Un sistema de pensamiento es el conjunto organizado de principios, conceptos, métodos y supuestos que una persona o comunidad usa para interpretar la realidad, tomar decisiones y generar conocimiento.

## Tres tesis fundamentales

### 1. Los sistemas de pensamiento determinan qué conocimiento es posible

No solo organizan lo que sabemos — definen lo que PODEMOS saber. La pregunta más poderosa: ¿qué conocimiento es IMPOSIBLE con nuestros sistemas actuales?

### 2. Si no puede criticarse, no es sistema de pensamiento. Es dogma.

Un sistema poderoso invita a ser criticado. HA debe ser falsable — deben existir condiciones bajo las cuales demostremos que está equivocado.

### 3. ¿Quién define los sistemas de pensamiento?

Históricamente: filósofos, científicos, civilizaciones completas. Ahora emerge una cuarta posibilidad: IA + sistemas humanos híbridos. Eso es lo que HA está construyendo.

## Datos vs. Conocimiento en TaaS

| | Datos | Conocimiento |
|---|---|---|
| **Propiedad** | Privados — del cliente | Perdura — sobrevive al proyecto |
| **Portabilidad** | Removibles — el cliente los lleva | Se comparte — beneficia sin exponer datos |
| **Ciclo de vida** | Olvidables — se pueden borrar | Crece — cada interacción lo alimenta |

### Pipeline de memoria

```
Working Memory (sesión) → Project Memory (persistente) → Organizational Memory (anonimizado, compartido)
```

### Intención como ancla

HA captura la intención con el Legado. Todo dato y todo conocimiento existen porque hubo una intención que les dio sentido.

```
Legado → Intención → Datos → Conocimiento
```

## La Matriz de Interacción

V = M × I es un filtro de valor, no un modelo de interacción. Lo que falta:

- Las 6 dimensiones no son independientes — se afectan mutuamente
- Una decisión en Technology impacta Community y Learning
- Sin modelo de interacción, las dimensiones son silos

### La solución

- Cada dimensión tiene relaciones con las otras 5
- Cada relación tiene una pregunta generadora
- 6 × 5 = 30 interacciones posibles
- El motor de contradicción detecta inconsistencias

TaaS implementa esta matriz como API: cada agente dimensional genera preguntas a las otras dimensiones automáticamente.
